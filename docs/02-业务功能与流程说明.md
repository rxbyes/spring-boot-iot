# 02 业务功能与流程说明

更新时间：2026-03-18

## 1. 业务域划分

平台按“接入-预警-处置-治理”划分为三大业务域：

1. 设备接入域：产品、设备、协议、消息、属性、下行指令。
2. 风险预警处置域：告警、事件、工单、风险点、阈值规则、联动规则、应急预案、分析报表。
3. 系统治理域：组织、用户、角色、菜单、区域、字典、通知渠道、审计日志。

## 2. 当前可用能力清单

### 2.1 设备接入域

- 产品新增/查询。
- 设备新增/查询（按 `id`、按 `deviceCode`）。
- HTTP 模拟上报。
- MQTT 上行接入（标准 Topic + `$dp` 兼容）。
- `mqtt-json` 协议解码。
- 消息日志持久化与查询。
- 最新属性更新。
- 设备在线状态更新。
- 下行发布最小能力（MQTT publish）。
- 消息追踪（按 TraceId/设备/Topic 检索）。

### 2.2 风险预警处置域

- 告警中心：列表、详情、确认、抑制、关闭。
- 事件处置：列表、详情、派发、接收、开始、完成、反馈、关闭。
- 风险点管理：CRUD、设备绑定/解绑。
- 阈值规则：CRUD。
- 联动规则：CRUD。
- 应急预案：CRUD。
- 报表分析：风险趋势、告警统计、事件闭环、设备健康。

### 2.3 系统治理域

- 组织管理（树结构 + CRUD）。
- 用户管理（CRUD + 密码重置）。
- 角色管理（CRUD + 用户角色关系查询）。
- 菜单管理（树 + 列表 + CRUD）。
- 区域管理（树 + 列表 + CRUD）。
- 字典管理（字典 + 字典项）。
- 通知渠道管理（CRUD + 渠道测试）。
- 审计日志管理（业务审计 + system_error 统计）。

## 3. 核心业务流程

### 3.1 设备接入主流程（HTTP）

1. 创建产品与物模型。
2. 创建设备并绑定产品。
3. 设备经 `/message/http/report` 上报。
4. 协议层标准化上报消息。
5. 写入消息日志与最新属性。
6. 更新设备在线/最近上报时间。

### 3.2 设备接入主流程（MQTT）

1. 设备按标准 Topic 或 `$dp` 上报。
2. 解析 Topic、鉴别直连/子设备/兼容模式。
3. 协议适配器完成解码、安全校验、必要解密。
4. 写入日志与属性，刷新在线状态。
5. 异常链路记录为 `system_error` 审计，并可触发 webhook 通知。

### 3.3 告警与事件闭环流程

1. 规则触发告警（`iot_alarm_record`）。
2. 告警确认/抑制/关闭。
3. 必要时转事件（`iot_event_record`）并派发工单（`iot_event_work_order`）。
4. 工单执行并反馈，事件关闭形成闭环。
5. 报表聚合输出闭环指标。

### 3.4 系统治理流程

1. `POST /api/auth/login` 登录，前端持久化 token。
2. 后端按用户角色返回动态菜单与按钮权限。
3. 前端请求自动附加 `Authorization: Bearer <token>`。
4. 业务页面按权限渲染按钮并调用对应接口。
5. 审计过滤器记录操作结果，失败和异常进入排障链路。

## 5. 角色与页面分区（前端）

- 一级分区：设备接入、风险处置、系统管理。
- 典型页面：
  - 设备接入：`/products`、`/devices`、`/reporting`、`/insight`、`/file-debug`、`/message-trace`、`/system-log`
  - 风险处置：`/alarm-center`、`/event-disposal`、`/risk-point`、`/rule-definition`、`/linkage-rule`、`/emergency-plan`、`/report-analysis`
  - 系统管理：`/organization`、`/user`、`/role`、`/menu`、`/region`、`/dict`、`/channel`、`/audit-log`
