# 15 前端优化与治理计划

> 文档定位：长期生效的前端治理规则、开放问题和下一步优化方向。
> 适用角色：前端研发、全栈研发、设计协作、代码审阅者、AI 前端助手。
> 权威级别：一级权威。
> 上游来源：`spring-boot-iot-ui` 当前实现、[06-前端开发与CSS规范.md](./06-前端开发与CSS规范.md)、近期前端改造结论、归档快照。
> 下游消费：页面重构、风格收口、交互评审、技能文档、前端任务模板。
> 变更触发条件：共享组件规则变化、页面风格收口、列表/抽屉模式调整、前端稳定性问题暴露。
> 更新时间：2026-03-22

## 1. 当前长期有效规则

1. 新页面优先复用 `PanelCard`、`StandardPagination`、`useServerPagination`、`StandardTableToolbar`、`StandardTableTextColumn`、`StandardDetailDrawer`、`StandardFormDrawer`、`StandardDrawerFooter`、`confirmAction` 和共享列表样式。
2. 页面配色继续统一到 `tokens.css` 与 `element-overrides.css` 的品牌 token；禁止为单页重新发明另一套蓝橙紫主色。
3. 工作台、抽屉、确认交互和概览卡必须保持“浅背景 + 白卡 + 轻描边 + 轻阴影”的控制台语法，不回退到高饱和营销式视觉。
4. 顶部壳层、分组导航、命令面板、通知中心、帮助中心继续收口到共享 shell 编排，不新增页面私有壳层能力。
5. `/in-app-message` 的桥接效果运营继续保持“同页折叠区 + 独立桥接详情抽屉”结构，复用共享分页、详情抽屉和表格列模式，不新增第二套桥接治理页面。
6. 新增页面和组件继续遵守 [06-前端开发与CSS规范.md](./06-前端开发与CSS规范.md) 中的“路由页 / 壳层编排 / 共享标准件 / 业务包装层”分层与命名规则，不回退到无语义命名。
7. 当前前端最小工程门禁继续保持 `npm run build`、`npm run test`、`npm run style:guard`；若后续接入 ESLint / Stylelint，需要先更新 `06` 与本文件。

## 2. 页面结构治理重点

### 2.1 列表页

- 默认结构保持为 `概览或标题区 -> 筛选区 -> 工具条 -> 列表 -> 分页`。
- 同一台账优先收口为一张主卡，避免“说明卡 + 列表卡 + 分页卡”层级过深。
- 已提交筛选条件必须有显式反馈，支持单项移除和一键清空。
- 平台治理和风险处置类列表继续沿用统一操作条、长文本省略和选择态计数，不回退到页面私有工具条。

### 2.2 详情与抽屉

- 详情抽屉优先采用“摘要秒开 + 后台静默补数”的体验，不把正常刷新状态包装成红色错误提示。
- 详情抽屉内的加载、空态、错误态必须与 `StandardDetailDrawer` 统一，不为单页单独造一套提示组件。
- 表单抽屉继续复用 `StandardFormDrawer` 和 `StandardDrawerFooter`，避免重复底部按钮样式与关闭逻辑。
- 桥接详情抽屉继续采用“摘要秒开 + 消息原文与尝试明细并行补数”的模式；尝试明细若为空或失败，只在抽屉内部表达，不影响上层治理列表继续使用。

### 2.3 链路验证中心

- `/reporting` 保持“左侧模拟上报、右侧诊断复盘”的双主卡结构。
- `deviceCode` 继续作为唯一主查询入口；`productKey`、`protocolCode`、`clientId` 保持只读派生。
- JSON / XML 格式化仅服务编辑体验；只有可识别 JSON 才参与协议帧判断。

## 3. 稳定性与质量规则

1. 控制台 warning 需要区分“单次巡检采样噪音”和“可稳定复现问题”，只有复测成立的问题才进入活跃治理范围。
2. `v-permission` 不得直接挂在没有稳定原生根节点的组件上；组件型操作菜单优先在脚本层过滤 action 列表。
3. 移动端侧栏关闭前必须先移出焦点，并只在桌面折叠态使用 tooltip 包裹导航图标。
4. 浏览器自动化与视觉回归的失败证据继续保留在历史台账中，但长期有效规则要回写到本文件和 [06-前端开发与CSS规范.md](./06-前端开发与CSS规范.md)。
5. 桥接响应摘要、目标摘要、帮助搜索高亮等消费态文本继续统一走安全文本切片渲染，不回退到 `v-html` 直出。

## 4. 当前开放问题

### P1

- 持续扫描共享页面中组件节点上的权限指令、分页漂移和详情抽屉私有样式回流风险。
- 继续控制开发态 warning 和非业务型控制台噪音，避免掩盖真实缺陷。
- 持续补齐前端静态门禁，优先评估 ESLint / Stylelint 与现有 `style:guard` 的协同方案。

### P2

- 明确视觉回归在日常提测流程中的准入等级。
- 继续把一次性页面优化经验沉淀为共享组件或治理规则，而不是继续累积页面私有实现。
- `站内消息` 桥接运营 V1 仍未覆盖导出、手工重试、趋势图表与更丰富的渠道洞察；后续应基于当前页内专区渐进增强，不再额外拆管理子系统。

## 5. 近期优先方向

1. 继续围绕“共享组件接管 + 页面私有样式回收”推进收口。
2. 把高价值页面的详情、表单、空态和导出行为进一步统一。
3. 将新的稳定规则优先更新到 [06-前端开发与CSS规范.md](./06-前端开发与CSS规范.md)；本文件只保留治理结论、开放问题和下一步方向。
