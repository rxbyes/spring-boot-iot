# Codex 执行规则

## 基本约束
- 项目名必须为 spring-boot-iot
- 包名必须为 com.ghlzm.iot
- 一期保持模块化单体
- spring-boot-iot-admin 为唯一启动模块

## 模块边界
- protocol 不要承担业务入库
- message 只做接入和分发
- device 承担设备核心业务逻辑
- framework 负责配置和基础设施

## 开发优先级
1. 设备上报主链路
2. 产品管理
3. 设备管理
4. MQTT 真接入
5. 规则 / 告警 / OTA

## 输出要求
- 先给出计划
- 列出变更文件
- 说明运行方式
- 标注未完成项

## 前端页面/样式变更附加要求
- 修改 `spring-boot-iot-ui` 页面、样式、文案、占位符、按钮文本时，必须确认文件内容保持 UTF-8 可读，不得把终端乱码直接写入源码。
- Windows 终端查看前端文件前，先切换到 UTF-8 口径再检查内容；修改后再次核对中文文案，重点排查 `鍒�`、`褰�`、`璇�`、`鐢�` 一类乱码残留。
- 列表页、分页区、筛选区优先复用既有标准能力：`PanelCard`、`StandardPagination`、`useServerPagination`、全局共享列表样式；不要在新窗口中重复造一套局部分页和列表样式。
- 如果页面优化过程中暴露出新的样式统一问题、编码问题、构建问题或操作风险，需要同步更新 `docs/15-前端优化与治理计划.md`，把现象、原因和防呆规则写清楚。
