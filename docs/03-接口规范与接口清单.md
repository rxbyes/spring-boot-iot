# 03 接口规范与接口清单

更新时间：2026-03-19

本清单以 Controller 源码为准，目标是作为二次开发与联调的主索引。

## 1. 通用规范

### 1.1 基础协议

- 接口风格：REST + JSON。
- 统一返回：`R`，典型格式：

```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

- 业务错误：`code != 200`，前端统一按失败处理。

### 1.2 鉴权规范

- 默认要求：`Authorization: Bearer <token>`。
- 公共白名单：
  - `POST /api/auth/login`
  - `POST /message/http/report`
  - `/api/cockpit/**`
  - `/actuator/**`
  - `/doc.html`
  - `/swagger-ui/**`
  - `/swagger-ui.html`
  - `/v3/api-docs/**`

### 1.3 链路追踪

- 请求/响应可携带 `X-Trace-Id`。
- `/api/*`、`/message/*` 会注入或透传 traceId。

### 1.4 分页约定

【根据代码推断】多数分页接口采用 `pageNum`、`pageSize`，返回 `PageResult`（`records`、`total`、`pageNum`、`pageSize`）。

## 7. 兼容与注意事项

- `iot_message_log` 为兼容视图，消息日志真实写入 `iot_device_message_log`。
- 部分“更新/删除”接口仍采用 `POST`（如风险点、规则、预案），属于当前历史风格，重构时需评估兼容性。
- `【待确认】` OpenAPI 文档是否与所有控制器同步生成且可用于外部 SDK 自动化生成。
