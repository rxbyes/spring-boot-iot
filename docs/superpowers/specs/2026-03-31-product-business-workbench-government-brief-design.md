# 产品经营工作台首长简报式视觉进化设计

> 日期：2026-03-31
> 范围：`spring-boot-iot-ui`
> 主题：在既有扁平化产品经营工作台基础上，继续进化为“首长简报式”的政企经营工作台
> 状态：设计已确认，待进入实现

## 1. 背景

`产品定义中心` 的统一产品经营工作台已经完成第一轮扁平化重构，解决了以下问题：

- `详情 / 物模型治理 / 关联设备 / 编辑治理` 已统一进入一个工作台抽屉
- 行内操作已收口为 `进入工作台 + 删除`
- 内页不再是旧版多套对象抽屉并行存在

但当前效果仍主要停留在“结构被压平”，还没有形成足够稳定的政企经营气场。用户进一步指出：

- 页面虽然统一了，但还不够大气
- 标题、留白、区块关系和阅读路径还不够像正式经营简报
- 各内页之间仍有轻微“组件拼装感”，缺少一套更强的上位排版秩序

因此，这一轮不再解决功能问题，而是继续提升视觉语言与版式气场。

## 2. 目标

- 在现有扁平化骨架上，升级为“首长简报式”的政企经营工作台。
- 强化标题层级、章节编排和留白节奏，让页面更稳、更克制、更有汇报气场。
- 让四个内页共享同一套“定调头部 + 章节主板块 + 节内主舞台”的阅读秩序。
- 保留统一工作台、统一页签、统一组件基线，不回退成私有大屏或营销风设计。

## 3. 非目标

- 不新增路由，不拆出独立详情页。
- 不新增后端接口或聚合字段。
- 不引入新的品牌主色体系，不做蓝紫科技风、深色驾驶舱风或营销展陈风。
- 不回流厚 hero、重渐变、大面积说明墙和第二套工作台骨架。

## 4. 用户已确认方向

本轮已确认：

- 主轴：`A 排版气场`
- 气质：`政企经营简报式`
- 落地方案：`A. 首长简报式`

解释固定为：

- 重点提升“大标题压场、章节主次、留白秩序、阅读威压感”
- 不追求花哨视觉，不靠色块堆砌气场
- 仍以统一工作台为容器，但内部要更像正式经营汇报材料

## 5. 总体设计原则

### 5.1 标题先定调

每个内页都必须有足够明确的“先定调，再读数，再落治理”结构：

- 一级：工作台头部的产品名与状态判断
- 二级：当前视图的章节标题
- 三级：节内的主舞台标题或台账标题

标题职责必须明确，避免当前“标题很多，但没有主次压差”的问题。

### 5.2 留白作为秩序，不是空

页面要通过统一的上下间距、区块内边距和列间距来建立稳定感：

- 头部与正文拉开更明确的呼吸距离
- 大区块之间用更完整的段落间距
- 节内卡片数量减少，但每块更完整、更有落点

### 5.3 章节化，而不是卡片化

这轮不继续增加卡片，而是减少“同权重卡片平铺”，改为“章节主板块 + 辅助信息”：

- 每一屏只保留一个主舞台
- 辅助指标只承担陪衬，不抢标题
- 治理提示要更像结论和批注，不像漂浮说明块

### 5.4 协调的流线

从头部到正文的阅读顺序固定为：

1. 产品身份与当前状态
2. 当前视图的核心章节标题
3. 首屏唯一主舞台
4. 支撑判断的辅助信息
5. 治理或台账主区

这样能把“工作台”从零散阅读变成有顺序的经营汇报阅读。

## 6. 统一工作台壳层

### 6.1 头部

统一工作台头部继续保留：

- 产品名称
- 产品 Key
- 状态 badge
- `productKey / 协议 / 节点 / 数据格式`
- 四个真实页签

但本轮要升级为：

- 标题字号更有压场感
- `产品经营工作台` 眉题更轻，但更像章节索引
- 状态 badge 更像“当前判断”，而不是普通标签
- meta chips 与 tabs 的节奏更规整，形成“简报页眉”感

### 6.2 视图壳层

统一工作台正文容器继续保留一个总 `view-shell`，但要把：

- 边框更轻
- 圆角更稳
- 背景更干净
- 页签与头部之间的节奏更顺

目标是让用户先感受到“这是一个完整的经营工作台页面”，而不是抽屉里嵌了 4 个组件。

## 7. 各视图落地

### 7.1 经营总览

总览页仍是整套工作台的主舞台，这轮要最先“立住气场”。

结构固定为：

1. 章节头：`设备规模与经营判断`
2. 主舞台：只保留一个大数与结论短句
3. 辅助指标列：在线设备、30 日活跃、平均在线时长
4. 经营判断板块：趋势摘要、接入提示
5. 契约与档案板块
6. 维护治理板块

精修重点：

- 主数字更像汇报中的“核心经营数据”
- 辅助指标退到次级阅读层
- “趋势摘要”“接入提示”“当前建议”改成更像经营判断结论
- 契约和档案不再显得像散列字段列表，而要更像附录摘要

### 7.2 物模型治理

物模型页需要从“治理组件页”提升为“治理简报页”。

结构固定为：

1. 章节头：`物模型治理`
2. 摘要条：真实证据概览、属性候选、待人工确认、正式模型
3. 主工作区：左目录、中主列表、右治理轨道

精修重点：

- 模式切换更像正式页签，不像切换按钮
- 摘要条更像概览条目，不像重复摘要卡
- 中间主列表更像“当前正文”，左右两侧退为辅区
- `进入完整治理` 按钮要成为清晰但克制的主动作

### 7.3 关联设备

关联设备页要从“概览 + 表格”提升为“先定调，再核账”的台账简报。

结构固定为：

1. 章节头：`设备运行概览`
2. 指标带：设备总数、在线设备、离线设备、在线比例
3. 台账主区：设备列表

精修重点：

- 指标带变成更统一的摘要横带，不再像几块轻卡片散落
- 台账标题与表格成为真正的主体
- 表头、空态和表格包裹层要更稳，不要显得轻飘

### 7.4 编辑治理

编辑页要从“编辑表单页”提升为“变更申请页”的气质。

结构固定为：

1. 章节头：`编辑治理`
2. 档案摘要条：产品 Key、协议编码、节点与状态
3. 内联治理提示
4. 表单主区：基础档案、接入基线、补充说明

精修重点：

- 章节头必须先定调“当前在做什么变更”
- 摘要条更像“变更上下文”
- 表单分区标题更正式，弱化普通表单感
- 保存区保持克制，不做大面积强视觉操作条
