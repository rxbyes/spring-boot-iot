# 产品定义中心到设备资产中心的高频台账模板迁移设计

> 日期：2026-03-31
> 范围：`spring-boot-iot-ui`
> 主题：从 `产品定义中心` 提炼高频台账工作台模板，并应用到 `设备资产中心` 主列表页
> 状态：设计已确认，待书面 spec 审阅

## 1. 背景

`产品定义中心` 在最近一轮重构后，已经形成一套相对稳定的高频台账工作台语法：

- `StandardPageShell + StandardWorkbenchPanel` 的页级节奏已经稳定
- 查询区、已生效筛选条、工具条、结果区、分页区的层级关系已经清晰
- 桌面表格和移动卡片形成了同一套阅读顺序
- 行内操作已统一为更紧凑的间距基线，并接入自适应操作列宽度

相比之下，`设备资产中心` 虽然已经部分复用共享组件，但仍存在以下问题：

1. 工作台骨架虽然相近，但筛选区、工具条、结果区的视觉节奏还没有完全进入同一套模板。
2. 设备页的结果区底板、loading 骨架、移动卡片和表格细节，仍保留较多页面私有语法。
3. 设备页的操作列、meta 标签和移动卡片字段密度与产品页尚未形成统一规范。
4. 当前优化如果只停留在单页微调，后续其他模块仍会重复“产品页一套、设备页一套”的分叉问题。

因此，本轮目标不是简单把设备页改得“更像产品页”，而是把 `产品定义中心` 已经验证过的设计语言提炼成一套可复用的高频台账模板，再让 `设备资产中心` 按其业务密度继承。

## 2. 目标

- 从 `产品定义中心` 提炼高频台账工作台模板，而不是继续在页面内复制局部样式。
- 把这套模板优先应用到 `设备资产中心` 主列表页。
- 统一两页的间距、字号、布局节奏、操作列间距、结果区底板和移动端卡片语法。
- 保留设备页更高的信息密度、更多筛选位和更复杂的状态表达，不把设备页压扁成产品页的翻版。
- 为后续 `异常观测台`、`链路追踪台` 等同类高频台账页的继续迁移提供模板基础。

## 3. 非目标

- 本轮不重构设备详情抽屉。
- 本轮不重构设备新增/编辑抽屉。
- 本轮不重构批量导入和更换设备流程。
- 不新增一级路由，不拆第二套设备列表页。
- 不修改设备 API、权限编码、分页协议和业务字段口径。

## 4. 用户已确认方向

本轮已在对话中确认以下约束：

- 范围：只处理 `设备资产中心` 主列表工作台，不把详情、编辑、批量导入、更换设备并入首轮。
- 路线：采用“方案 B”，即提炼 `产品定义中心` 模板，再适配到设备页。
- 风格强度：采用“同骨架同节奏，但保留设备页更高信息密度”的方向，而不是把设备页完全压成产品页的复制品。

因此，本轮不是“做一个新的设备页设计方案”，而是“定义并落地一套高频台账模板”。

## 5. 模板边界

本轮要抽出的模板边界如下。

### 5.1 模板首轮统一的内容

- 页壳与主工作台节奏：
  - `StandardPageShell`
  - `StandardWorkbenchPanel`
  - 标题、说明、筛选区、工具条、结果区、分页区之间的垂直留白
- 列表工作台语法：
  - 查询区
  - 已生效筛选条
  - 工具条
  - 结果区
  - 空态、加载态、分页
- 视觉节奏：
  - 间距
  - 字号
  - 卡片内边距
  - 结果区底板
  - 表头亮度
  - 移动卡片密度
  - 操作列和卡片操作区的按钮间距
- 交互节奏：
  - 桌面表格为主视图
  - 移动卡片为补充视图
  - 操作列自适应宽度
  - 同一套操作按钮语法

### 5.2 本轮不纳入模板的内容

- 设备详情抽屉结构
- 设备编辑表单结构
- 批量导入
- 更换设备
- 设备页业务流程与数据契约

### 5.3 设备页保留的业务差异

- 更多筛选字段与状态筛选维度
- `已登记资产 + 未登记上报名单` 的混合台账语义
- 设备特有字段，如在线、激活、登记、固件、网关/父设备关系
- 设备特有权限动作与业务动作

## 6. 模板页面语法

### 6.1 页面头部

设备资产中心继续采用 `StandardPageShell + StandardWorkbenchPanel`，但与产品定义中心统一以下语法：

- 只保留一个主标题和一句轻说明
- 不新增说明墙、第二层标题或页内重复导航
- 标题、说明、筛选区之间的垂直留白统一跟随产品页

设备页保留“设备台账”定位，但进入与产品页同一页级节奏。

### 6.2 筛选区

筛选区继续复用 `StandardListFilterHeader`，但设备页允许保留更多首屏筛选位。

统一规则：

- 快速搜索仍是第一入口
- 其余筛选项按统一栅格排布
- 查询、重置、新增、批量导入使用同一按钮尺寸和动作组节奏
- 快速搜索标签、已生效筛选条的位置和样式与产品页一致

设备页保留更多筛选字段，但不再保留与产品页明显不同的筛选区密度和对齐方式。

### 6.3 工具条

工具条与产品页统一为轻量语法：

- 左侧只保留真正有效的元信息
- 右侧统一收口为 `更多操作 + 刷新列表`
- 元信息文字密度、间距和对齐方式跟随产品页

设备页当前的统计项会保留核心口径，但顺序要按“已选 / 已登记 / 在线 / 激活 / 停用”之类的阅读顺序收口，避免工具条过满。

### 6.4 结果区

结果区与产品页统一为同一底板语法：

- 轻底色
- 轻描边
- 统一 loading 蒙层
- 同一空态与结果区阅读顺序

设备页不再保留更散、更像另一套页面系统的结果容器表达。

### 6.5 桌面表格

桌面表格继续作为设备页主视图，但对齐产品页的模板规则：

- 表头亮度与表格层级统一
- 单元格字号与行高统一
- 长文本继续复用共享文本列语法
- 操作列继续使用共享自适应宽度
- 操作项间距统一到紧凑基线

设备页可保留更多状态列，但行内操作和表格节奏不再独立演化。

### 6.6 移动卡片

设备页移动卡片继续保留现有信息量，但进入与产品页一致的模板语法：

- 卡片内边距统一
- 标题字号统一
- 辅助字段字号统一
- meta 标签高度与留白统一
- 卡片内字段行距统一
- 卡片底部操作区按钮间距统一

设备页可以继续保留语义状态标签，但它们只承担业务表达，不再形成另一套视觉体系。

## 7. 实现方式

### 7.1 总体策略

本轮不引入一个很重的新“列表模板组件”去硬绑产品页和设备页，而是采用：

- 共享语法层
- 页面继承层

两层结构落地。

### 7.2 共享语法层

共享语法层承接以下内容：

- 主工作台节奏
- 结果区底板语法
- loading / empty / list 的统一节奏
- 表格与移动卡片的字号、间距、内边距和 meta 标签密度
- 操作列和卡片操作区的统一间距

共享语法层优先通过：

- 扩展现有共享组件
- 扩展现有全局列表样式或共享 token
- 回收产品页与设备页之间重复但分叉的样式实现

来完成，而不是再新增一套页面私有包装组件。

### 7.3 页面继承层

- `ProductWorkbenchView.vue` 继续作为模板来源页
- `DeviceWorkbenchView.vue` 继承这套模板语法

设备页只保留真正属于其业务的差异：

- 更多筛选位
- 更多状态维度
- 更高字段密度
- 设备专属业务动作

### 7.4 本轮优先改动层

- 设备页筛选区布局和动作区节奏
- 设备页工具条元信息数量与排列
- 设备页结果区底板、loading、empty、table、mobile card 的样式骨架
- 设备页桌面操作列和移动卡片操作区的间距
- 设备页 scoped CSS 中与模板重复但分叉的样式

### 7.5 本轮不改动层

- 设备详情抽屉
- 设备编辑抽屉
- 批量导入/更换设备流程
- API、路由、权限、分页和设备业务逻辑

## 8. 验收标准

本轮通过的标准不是“设备页长得像产品页”，而是“设备页进入与产品页同一套高频台账模板系统”。

### 8.1 视觉与布局

- 设备资产中心主列表页的标题、说明、筛选区、工具条、结果区、分页区整体节奏与产品定义中心一致
- 桌面表格与移动卡片的字号、间距、按钮间距、meta 标签密度进入同一规范
- 操作列右侧不再出现明显大块空白
- 设备页不再保留与产品页明显分叉的结果区底板、loading 骨架和卡片节奏

### 8.2 交互与自适应

- 保持桌面表格主视图、移动卡片补充视图
- `720px` 左右断点切换稳定
- 不新增手动“表格/卡片视图”切换
- 筛选、分页、查询、重置、刷新、新增设备、批量导入不发生行为回退
- 操作列与卡片操作区继续支持自适应，不出现裁切、换行错乱和按钮堆叠异常

### 8.3 业务不回退

- 保留设备页现有筛选字段
- 保留当前台账字段与状态口径
- 保留已登记/未登记混合语义
- 保留现有权限动作与业务动作

## 9. 回归与验证

### 9.1 页面回归范围

- `设备资产中心` 主列表页
- `产品定义中心` 主列表页
- 已接入共享操作列宽度/间距策略的相关页面，至少进行静态回归确认，避免模板化时把现有效果打散

### 9.2 工程验证

前端实现完成后至少执行：

- 受影响组件/视图测试
- `npm run component:guard`
- `npm run list:guard`
- `npm run style:guard`
- `npm run build`

如测试范围已覆盖 `ProductWorkbenchView`、`DeviceWorkbenchView` 或相关共享组件，应优先跑受影响测试而不是只做全量盲跑。

## 10. 文档同步要求

本轮不只是页面优化，还要把模板规则原位沉淀到现有权威文档。

至少需要同步更新：

- `docs/06-前端开发与CSS规范.md`
- `docs/15-前端优化与治理计划.md`
- `docs/08-变更记录与技术债清单.md`

文档需要明确写出：

- 产品定义中心已成为高频台账模板来源页之一
- 设备资产中心已按同一模板语法收口
- 后续同类高频台账优先按“抽模板再迁移”的方式推进，而不是继续单页微调

## 11. 风险与控制

### 11.1 风险

- 过度复制产品页，导致设备页业务密度下降
- 只做设备页局部微调，导致模板目标落空
- 新增过重的共享包装组件，反而让产品页和设备页耦合变重

### 11.2 控制

- 保持“同骨架同节奏，但保留设备页更高信息密度”的约束不变
- 优先扩现有共享组件和共享样式，而不是新建大而重的模板组件
- 只处理主列表工作台，不把详情、编辑、批量导入、更换设备强行并入首轮

## 12. 预期结果

完成本轮后，`产品定义中心` 与 `设备资产中心` 会进入同一套高频台账工作台模板体系：

- 页面看起来属于同一个产品系统
- 设备页仍保持自身业务密度
- 操作列、字号、间距、结果区、移动卡片进入同一规范
- 后续其他高频台账页可以沿这套模板继续迁移，而不再从单页样式微调起步
