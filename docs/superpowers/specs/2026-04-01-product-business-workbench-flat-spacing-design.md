# 产品经营工作台极简扁平化细节进化设计

> 日期：2026-04-01
> 范围：`spring-boot-iot-ui`
> 主题：在既有“首长简报式”产品经营工作台基础上，继续做细节层面的极简扁平化收敛
> 状态：设计已确认，待用户审阅 spec

## 1. 背景

`/products` 的统一 `产品经营工作台` 已完成以下两轮工作：

- 第一轮：统一 `详情 / 物模型治理 / 关联设备 / 编辑治理` 四个入口，收口到一个工作台抽屉
- 第二轮：按“首长简报式”收紧标题、页眉、概览阅读顺序与文案密度

但最新验收仍指出几个明显问题：

- 详情页仍然“框太多”，趋势摘要与契约档案像多个小盒子拼在一起
- 页眉右侧指标仍保留明显卡片感，扁平化不够彻底
- 上下文距离缺少节奏，存在“高的高不起来、低的低不下去”的问题
- 页面仍主要靠边框和浅底分块，不是靠留白、对齐、字号梯度和细线建立层级
- `趋势摘要` 与 `接入契约与产品档案` 虽然已经分离语义，但视觉上仍像被一个大盒子裹住

因此，这一轮不是再改信息结构，而是对同一套内容做“去框、去装饰、重做节奏”的细节进化。

## 2. 目标

- 把 `产品经营工作台` 继续收成更极简、更平、更有审美节奏的详情页
- 用留白、对齐、字号和细线替代大部分卡片边框和包围盒
- 让页眉成为一条横向经营带，而不是“左一块 + 右一堆卡片”
- 让 `趋势摘要` 与 `接入契约与产品档案` 变成两个独立章节，而不是被一层总框包住的组合模块
- 让整页建立清晰的“高的高，低的低”的间距体系

## 3. 非目标

- 不新增接口、字段或统计口径
- 不调整 `/products` 主列表逻辑和工作台开闭逻辑
- 不改变四个页签的业务语义与路由结构
- 不回退到厚卡片、Hero Stage、说明墙、治理侧栏等旧语法
- 不引入新的品牌色或营销化装饰语言

## 4. 用户确认结论

本轮设计确认过程已经完成，结论固定如下：

- 视觉方向：`C 方案`
- 解释：`更极简扁平，尽量去框去线，主要靠间距和文字层级撑住版式`
- 作用范围：`D`
- 解释：`不是只改局部，而是整个详情页一起压平`

用户已确认的关键诉求包括：

- 去掉无用的框
- 趋势摘要与契约档案模块可以拆成两个清晰章节
- 重新建立页面的上下文距离和层级节奏
- 让页面更有审美，不要所有块都同一个高度、同一种“组件感”

## 5. 设计总原则

### 5.1 扁平化不是无层级

本轮的“扁平”不是取消一切差异，而是把层级从“盒子感”转移到：

- 标题大小
- 数字大小
- 行高与留白
- 分隔线强弱
- 章节之间的段落感

结果应该是页面更平，但阅读秩序更强。

### 5.2 少框，多分段

当前问题不是信息太多，而是每层都在自己画边框。  
本轮要求：

- 外层大框能删则删
- 内部单元格不再各自成卡
- 尽量只保留章节级别的 very light 容器和内部细分隔线

阅读上要像一份整理过的经营页，而不是组件库样例页。

### 5.3 留白分三级

整页间距必须显式分成三档：

- `一级留白`：页眉与正文、章节与章节之间
- `二级留白`：章节标题到底下内容之间
- `三级留白`：字段标签和值、单元内部元素之间

禁止再出现所有模块上下内边距几乎一样的情况。

### 5.4 高的高，低的低

页面中不同信息必须有不同重心：

- 页眉最高，负责压场
- 趋势摘要最轻，负责快速浏览
- 契约档案最稳，负责信息收口

短信息不能被做成和长说明同样厚重的盒子；长说明也不能被硬塞进和短指标一样的格子高度。

## 6. 页眉重排设计

### 6.1 结构

`ProductBusinessWorkbenchDrawer` 的页眉改成“一条横向经营带”，不再是“左侧标题块 + 右侧一组明显卡片”。

页眉固定拆成三层：

1. 左侧标题区
2. 右侧横向数据带
3. 下方元信息串 + 真页签

### 6.2 左侧标题区

只保留：

- `产品经营工作台`
- 产品名

产品名下方不再使用鼓起的 tag/chip 视觉，而改成一条轻量元信息行。  
`productKey / 协议 / 节点 / 数据格式` 改为串联式信息，不再每项都像独立胶囊。

### 6.3 右侧指标带

保留当前 3 个核心读数：

- `关联设备`
- `在线设备`
- `30 日活跃`

但呈现改成：

- `关联设备` 为主指标，字号最大
- `在线设备`、`30 日活跃` 为副指标，字号次一级
- 三者之间只用细竖线、极浅底或 very light 局部底色区分
- 不再让三个指标各自成为厚卡片

目标是“经营数据带”，不是“概况卡片组”。

### 6.4 页签

页签必须从“胶囊按钮排布”继续压平为“文字 + 底线”的扁平 rail：

- 默认态主要靠文字颜色区分
- 激活态只保留细底线和品牌色文字
- 不再保留明显 pill 感和浮起阴影

## 7. 经营总览正文重排

### 7.1 从一个总框，改成两个章节

`ProductDetailWorkbench` 当前的 `趋势摘要与契约档案` 总容器需要继续拆解。

视觉上改成两个独立章节：

- `趋势摘要`
- `接入契约与产品档案`

不再使用一个外层总框把两个章节包在一起。  
章节之间靠垂直留白和标题层级分开，而不是大盒子。

### 7.2 趋势摘要

趋势摘要继续保留 `2 行 x 3 列`，但从“六个整格卡片”调整为“轻信息板”。

具体要求：

- 每个单元只保留 `标签 + 值`
- 不做整格包围边框
- 主要保留横向分隔线和 very light 列间分隔
- `趋势判断` 允许占用更舒展的文本节奏，但仍维持在同一网格中
- 数字信息不需要普遍加粗，只通过字号与位置建立主次

趋势摘要整体应更像一段被整理过的经营读数，而不是 6 张小卡片。

### 7.3 接入契约与产品档案

契约档案继续保留 `3 行 x 3 列`，但从“八个字段盒子”调整为“字段表”。

具体要求：

- 整块只有 very light 的底板，甚至在极简版本中可进一步淡化为纯平底
- 主要依赖行分隔
- 列与列之间只用极细竖线，不做完整单元格描边
- `产品说明` 继续占两列
- `产品状态` 与 `产品说明` 同行
- `产品说明` 这一行允许自然更高，形成清楚的节奏落差

目标不是“九宫格”，而是一张高级字段表。

## 8. 字体、线条与灰度规则

### 8.1 字号层级

全页只保留 4 档核心层级：

- `产品名`
- `主指标数字`
- `章节标题 / 副指标数字 / 重点字段值`
- `标签 / 元信息 / 普通字段值`

不要再通过大面积 `font-weight: 700` 伪造层级。

### 8.2 字重

只允许：

- 产品名明显更重
- 主指标数字明显更重

其余字段值和趋势数值尽量回到正常字重或只微提一档。  
这样页面会更平，也更像政企经营页，而不是所有值都在抢主舞台。

### 8.3 线条

全页只保留三种线：

- `章节分隔线`：最弱
- `表内行线 / 列线`：中弱
- `重点数据带分隔线`：最清楚

不允许再出现：

- 每个单元都画一圈
- 外层一圈、内层再一圈、单元里再一圈

### 8.4 阴影与底色

阴影继续收掉：

- 页眉允许保留 very weak 阴影或层次
- 正文两段尽量不依赖阴影
- 底色必须更浅、更平

不要同时使用：

- 明显描边
- 明显底色
- 明显阴影

这三者最多选一到两项，且都要压弱。
