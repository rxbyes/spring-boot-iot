# 产品经营工作台客户语言化设计

> 日期：2026-04-04
> 范围：`spring-boot-iot-ui`
> 主题：把 `/products` 产品经营工作台进一步收敛为客户可读的信息架构
> 状态：设计已确认，待实现

## 1. 背景

产品经营工作台已经完成一轮信息去重，但当前仍保留明显偏研发口径的表达：

- 页签仍使用 `经营总览 / 物模型治理 / 关联设备 / 编辑治理`
- `编辑治理` 仍与客户阅读型页签并列，打断“看产品、看设备、看契约”的主路径
- `物模型治理` 与内部模式文案仍偏工程实现视角，不够贴近客户认知

当前问题不在能力，而在命名和入口编排仍然更像研发后台，而不是客户使用的产品经营台。

## 2. 目标

- 把产品经营工作台收敛为客户一眼能懂的三段式结构
- 让编辑动作从页签区退出，回到头部次动作，减少阅读与编辑混杂
- 把 `物模型治理` 改写为更偏业务契约语义的客户表达
- 继续保持同一抽屉统一上下文、统一保活和统一回写

## 3. 非目标

- 不改变 `/products` 路由与后端接口
- 不引入新的页面或新的抽屉层级
- 不重做产品编辑表单结构
- 不改变契约治理 compare / apply 能力本身

## 4. 已确认方向

用户已确认采用以下客户视角收敛方案：

- 页签改为 `产品总览 / 关联设备 / 契约字段`
- 删除页签中的 `编辑治理`
- 在工作台头部动作区新增 `编辑档案`
- `契约字段` 页继续承接现有物模型治理能力，但页面文案改成客户更易理解的表达

## 5. 设计方案

### 5.1 工作台信息架构

- 保留统一 `产品经营工作台` 抽屉
- 固定三个客户页签：
  - `产品总览`
  - `关联设备`
  - `契约字段`
- `编辑档案` 不再作为并列页签，而是头部动作按钮

### 5.2 编辑入口

- 头部动作区新增 `编辑档案`
- 点击后仍进入原有编辑工作区，不新增新路由或新弹层
- 编辑完成或取消后，默认回到 `产品总览`

### 5.3 契约字段语义

- 原 `物模型治理` 页签对客户统一显示为 `契约字段`
- 设计器内部模式文案同步去工程化：
  - `手动提炼` 改为 `证据比对`
  - `正式模型` 改为 `正式字段`
- 其它说明继续保持简短，突出“字段确认”“证据核对”“正式契约”三层语义

### 5.4 统一上下文

- `detail / model / devices / edit` 仍在同一工作台上下文内切换
- 设备页继续按首次进入时加载设备清册
- `编辑档案` 虽不再出现在页签条，但仍允许通过既有内部编排进入同一编辑工作区

## 6. 测试与文档

  - 点击 `编辑档案` 后仍进入原有编辑工作区
