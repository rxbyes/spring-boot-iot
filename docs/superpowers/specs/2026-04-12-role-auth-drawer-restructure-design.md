# 角色权限抽屉分层重构设计

## 背景

当前 `spring-boot-iot-ui/src/views/RoleView.vue` 已把角色基础信息、菜单树、按钮权限、授权摘要都收口到同一个右侧抽屉内，但抽屉中的权限区仍沿用“一棵树同时展示目录、页面、按钮”的表达方式：

- 页面菜单和按钮节点混在同一个树层级里，用户很难快速判断“访问页面”与“操作按钮”的边界。
- 左侧基础信息区同时承担表单、授权摘要和长说明，右侧权限区又堆叠 `alert + 搜索 + 树`，整体扫读压力大。
- 当前主要操作路径是“先批量勾选页面范围，再精修少数页面按钮”，但现状要求用户在同一棵树里同时完成两类动作，效率低且容易误授。

本轮目标不是重做权限模型，而是在不改后端接口、不放大抽屉宽度的前提下，重构前端授权工作区的层级表达和交互路径。

## 目标

- 优先解决“页面菜单”和“按钮权限”混在一起看不清的问题。
- 在保持当前抽屉宽度的前提下，降低角色授权抽屉的拥挤感和扫读成本。
- 保留“先批量勾页面，再精修少量按钮”的高频操作路径。
- 保持现有后端 `menuIds` 提交语义、菜单真相来源和按钮权限颗粒度不变。
- 同步更新前端治理文档，防止角色权限页回流到“树里直接堆按钮”的旧模式。

## 非目标

- 本轮不新增后端接口、不修改 `sys_menu` / `sys_role_menu` 等数据库结构。
- 本轮不调整 `RoleView` 列表页骨架，不改动 `StandardWorkbenchPanel + StandardListFilterHeader + StandardTableToolbar + StandardPagination` 列表合同。
- 本轮不把角色授权抽屉升级为近全屏抽屉。
- 本轮不新增“跨全部页面一键全选所有按钮”的高风险操作。

## 方案对比

### 方案 A：三段式工作区，推荐

右侧权限区拆成：

1. `步骤 1：页面授权`
2. `步骤 2：已选页面`
3. `当前页面按钮权限`

页面树只展示目录和页面；按钮节点从树中移出，改为按“当前页面”单独展示。

优点：

- 最清楚地区分“页面访问范围”和“按钮操作颗粒度”。
- 最贴合当前高频路径：先勾页面，再精修少量按钮。
- 不依赖扩大抽屉宽度，也不需要频繁切标签页。

缺点：

- 需要在同一抽屉里维护三个权限子区块，前端状态组织会更细。

### 方案 B：步骤折叠式

先完成页面树授权，再展开第二步按钮配置。

优点：

- 结构更像向导，理解成本低。

缺点：

- 高频运维用户需要额外展开/切换步骤，连续操作效率偏低。
- 页面和按钮的关系不能稳定同屏展示。

### 方案 C：双标签页

使用 `菜单授权 / 按钮授权` 两个标签页，分别维护页面树和按钮列表。

优点：

- 单屏视觉最整洁。

缺点：

- 用户在“批量勾页面后立即精修按钮”时需要频繁切页签。
- 当前上下文容易丢失，不适合本任务的高频路径。

结论：采用方案 A。

## 设计约束

### 1. 权限真相来源保持不变

- 菜单与按钮权限仍来自后端动态菜单树和权限编码。
- `RoleView` 仍通过现有角色详情接口读取 `menuIds`。
- 提交时仍向角色新增/更新接口写回同一个 `menuIds` 数组。

### 2. 抽屉宽度保持当前基线

- 延续 `StandardFormDrawer` 当前宽度，不扩宽。
- 通过信息分层、摘要收口和子区块拆分解决拥挤，不靠“加宽抽屉”掩盖结构问题。

### 3. 按钮权限保留，不从权限模型删除

- 仅移除“按钮节点在树中的展示方式”。
- 按钮节点、按钮权限码、角色已有按钮授权都继续保留。
- 不同角色仍可基于同一页面授予不同按钮组合，不损失颗粒度。

## 交互设计

### 1. 抽屉总体布局

抽屉保持左右两栏：

- 左栏：角色基础信息 + 精简后的授权摘要
- 右栏：新的三段式授权工作区

右栏内部按纵向阅读节奏拆为：

1. `步骤 1：页面授权`
2. 下半区左侧 `步骤 2：已选页面`
3. 下半区右侧 `当前页面按钮权限`

### 2. 页面授权树

页面授权树只展示：

- `type = 0` 的目录
- `type = 1` 的页面

不再展示：

- `type = 2` 的按钮节点

规则：

- 目录节点只承担层级展示，不进入按钮授权语义。
- 用户勾选对象只保留页面节点。
- 页面树保留现有搜索、全选页面、清空页面能力。
- 页面树顶部说明收口为一条轻提示，不再在每个节点旁重复“目录自动补齐”等说明。

### 3. 已选页面列表

已选页面列表用于承接“按钮精修入口”，只展示当前已授权的页面。

每个页面卡片只表达：

- 页面名称
- 所属模块或路径摘要
- 按钮状态：
  - `未配置按钮`
  - `已选 X 个按钮`
  - `无独立按钮`

不在列表中展开全部按钮名，避免再次形成拥挤长列表。

### 4. 当前页面按钮权限区

当前页面按钮权限区只展示当前选中的页面下的按钮清单：

- 当前页内搜索
- 当前页按钮全选
- 当前页按钮清空
- 按钮逐项勾选

按钮清单使用“权限项列表”而不是树节点形式，每项固定展示：

- 勾选框
- 按钮名称
- 简短说明或业务语义
- 权限码

不做跨全部页面的一键按钮全选，避免误授。

## 状态与默认值规则

### 1. 编辑已有角色

- 页面授权与按钮授权都按后端已有 `menuIds` 正确回显。
- 若角色已有某页面下的按钮授权，则进入按钮区时保持原勾选状态。

### 2. 新增角色

- 新增角色首次打开时无页面、无按钮。
- 新勾选页面时，该页面按钮默认全不选。

### 3. 取消页面

- 一旦取消某个页面，该页面下按钮授权同步失效。
- 保存时不能继续把该页面下按钮 id 带回 `menuIds`。
- 如果当前按钮区正在编辑这个页面，则自动切换到下一个已选页面；若不存在下一个，则回到“请选择页面”空态。

### 4. 当前页面选择

- 默认优先保持最近一次操作的页面为当前页面。
- 用户点击“已选页面列表”中的任意页面时，按钮区切换到对应页面。
- 若页面无独立按钮，则按钮区展示“当前页面暂无独立按钮权限”空态，而不是空白区域。

### 5. 搜索边界

- 页面树搜索只匹配目录/页面名称、编码、路由。
- 按钮区搜索只匹配当前页面下按钮名称与权限码。
- 两套搜索互不影响。

### 6. 未保存关闭

- 若页面授权或按钮授权有未保存改动，关闭抽屉前给出统一确认提示。

## 视觉与文案收口

### 1. 左栏授权摘要

左栏不再展示长标签云，改成数字化摘要，例如：

- `已选页面 X`
- `已配置按钮页面 Y`
- `当前页已选按钮 Z`

左栏保留角色表单主任务，不再承担授权明细浏览。

### 2. 标题与说明

统一把原“菜单与按钮授权”拆成分步标题：

- `步骤 1：页面授权`
- `步骤 2：已选页面`
- `当前页面按钮权限`

这样用户一眼就能理解新的操作顺序。

### 3. 树节点减负

页面树节点只保留：

- 页面名称
- 类型标签
- 一行编码 / 路由

不再在单个节点内堆叠按钮标签、补齐提示和重复说明。

## 前端实现边界

### 1. 数据映射

前端需要基于完整菜单树派生三份视图数据：

1. 页面树数据：仅目录 + 页面
2. 页面到按钮的映射：`pageId -> buttonNodes[]`
3. 已选页面视图：根据当前 `menuIds` 中的页面节点生成

保存时需要把：

- 已选页面 id
- 各页面已选按钮 id

重新合并回原有 `menuIds` 提交结构。

### 2. 组件拆分建议

为避免 `RoleView.vue` 继续膨胀，授权区建议拆成最小局部单元：

- 页面树面板
- 已选页面列表
- 当前页面按钮面板

如果最终实现选择不拆独立组件，也至少要把数据派生和授权合并逻辑抽出为局部工具函数，避免模板与状态继续纠缠在一个大块里。

## 验证方式

### 1. 前端交互

- 旧角色打开时，页面和按钮授权能正确回显。
- 新角色新勾页面时，按钮默认全不选。
- 页面取消勾选后，对应按钮不会继续出现在保存结果里。
- 页面树只显示目录和页面，不再显示按钮节点。
- 按钮区只显示当前页面下的按钮，不跨页面混排。

### 2. 提交语义

- 保存 payload 继续沿用现有 `menuIds` 语义，不新增字段。
- 不破坏既有鉴权链路和 `/api/auth/me` 消费路径。

### 3. 文档与治理

- `docs/02` 说明角色权限页的授权路径由“同树勾选页面/按钮”更新为“页面树 + 按页按钮精修”。
- `docs/06` / `docs/15` 记录角色权限页新的分层交互规则，明确按钮节点不得再回流到树展示中。
- `docs/08` 记录本次页面结构变化和验证结果。

## 风险与缓解

### 风险 1：页面与按钮合并逻辑写错，导致已有按钮回显丢失

缓解：

- 以现有 `menuIds` 为唯一输入，先补测试锁定“旧角色回显”行为，再做 UI 重构。

### 风险 2：取消页面后按钮残留提交

缓解：

- 保存前统一基于“已选页面集合”过滤按钮 id，不允许孤立按钮进入最终 payload。

### 风险 3：在不加宽抽屉的情况下，下半区仍然拥挤

缓解：

- 已选页面列表只展示页面卡片摘要，不展开按钮。
- 按钮区固定为当前页面单列表达，不再同屏展开多个页面的按钮清单。
