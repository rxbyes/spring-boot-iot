# 角色权限抽屉全权限树重构设计

## 背景

`/role` 当前抽屉仍停留在上一版“页面树 + 已选页面 + 当前页面按钮”的拆分方案。该方案虽然把按钮从树里移走了，但仍然没有解决两类核心问题：

- 管理员无法在一个统一工作区里直观看到从一级目录到按钮的完整授权层级。
- 目录、页面、按钮之间的选择关系仍然割裂，导致“先选目录/页面，再按需细化按钮”的真实授权路径不顺手。

本轮目标是回到真实管理员授权模型：一个抽屉内覆盖一级菜单、二级菜单、页面、按钮四个层级，并保留按钮颗粒度控制，同时把视觉结构改得更扁平、更不拥挤。

## 目标

- 管理员可以分配所有权限层级：目录、菜单、页面、按钮。
- 左侧只保留一棵完整权限树，右侧只展示当前节点的直属子级明细。
- 顶部用摘要回答“总共授权了多少、当前节点是什么状态”，不再把长说明塞进树区域。
- 搜索只影响显示结果，不改变已选权限状态。
- 继续沿用现有后端 `menuIds` 合同，不新增接口或数据库结构。

## 非目标

- 不新增后端接口，不修改 `sys_role_menu`、`sys_menu` 结构。
- 不把角色授权抽屉改成全屏工作台。
- 不在树节点里回流路由、权限码、长说明等多行堆叠信息。
- 不删除按钮权限颗粒度，也不退回“只配页面不配按钮”的旧模型。

## 方案结论

采用统一的“全权限树 + 当前节点平铺详情 + 顶部摘要”方案。

### 结构

- 左侧：完整权限树
- 右侧：当前节点直属子级的平铺授权面板
- 顶部：授权摘要与当前节点状态

### 选择规则

- 勾选任意父节点时，默认勾选该节点及全部后代。
- 之后允许单独取消某些后代节点。
- 父节点在“自身已授权但子级未全选”或“子级部分授权”时，展示半选态。
- 未勾选父节点时，仍允许从右侧对子级单独授权；系统会自动补齐所需祖先节点，确保提交仍满足现有 `menuIds` 语义。

## 交互设计

### 1. 抽屉布局

抽屉保持左右两栏：

- 左栏：角色基础信息 + 精简授权摘要
- 右栏：权限工作区

权限工作区内部结构：

1. 顶部授权摘要条
2. 左侧权限树
3. 右侧当前节点直属子级列表 / 当前节点详情

### 2. 左侧权限树

树覆盖全部权限类型：

- `type = 0`：目录 / 菜单分组
- `type = 1`：页面
- `type = 2`：按钮

树节点行内只保留：

- 复选框
- 节点名称
- 类型标签
- 子级数量

不再在树行内堆叠：

- 路由
- 权限码
- 长说明
- 多层摘要标签

默认不全量展开，只展示收敛后的树结构；用户按需展开。搜索命中时仅影响显示结果，不会清除既有勾选状态。

### 3. 顶部摘要

摘要区直接回答当前授权情况，例如：

- 已授权目录数
- 已授权页面数
- 已授权按钮数
- 当前节点状态：未授权 / 已授权 / 部分授权
- 当前节点直属子级已授权数量

摘要只做数字化概览，不承载明细浏览。

### 4. 右侧当前节点平铺详情

右侧永远只看当前节点，不再同时展开多个层级。

#### 当前节点为目录 / 菜单时

右侧展示其直属子级（目录或页面）的平铺卡片/列表，每项支持：

- 勾选 / 取消授权
- 查看类型
- 查看路由或编码摘要
- 切换为新的当前节点

#### 当前节点为页面时

右侧展示该页面直属按钮列表，每项支持：

- 勾选 / 取消按钮权限
- 查看按钮名称
- 查看权限码或简短说明

#### 当前节点为按钮时

右侧不再展示空白列表，而是展示当前按钮的权限信息卡：

- 名称
- 权限码
- 所属父页面
- 当前授权状态

## 状态模型

### 1. 前端本地状态

前端统一维护一个“当前已授权节点 id 集合”，所有类型节点都进入同一个集合。

在此基础上派生：

- 节点映射
- 当前节点
- 每个节点的后代集合
- 每个节点的视觉状态：选中 / 半选 / 未选
- 右侧直属子级展示数据

### 2. 勾选行为

#### 勾选节点

- 将当前节点加入已授权集合
- 将全部后代节点加入已授权集合
- 自动补齐祖先节点，避免出现孤立子节点

#### 取消节点

- 将当前节点及全部后代从已授权集合中移除
- 祖先节点不自动强制移除；若管理员希望收回父节点权限，可继续手动取消父节点

该策略兼容“先选父节点，再取消某些按钮”的实际操作路径，也兼容“从未勾选父节点下直接勾选某个子节点”的细粒度操作。

### 3. 回显与提交

- 编辑已有角色时，直接基于角色详情返回的 `menuIds` 回显到本地授权集合。
- 保存时继续提交单一 `menuIds` 数组。
- 后端现有祖先补齐逻辑保持不变。

## 视觉收口

- 树与右侧详情区采用更扁平的单行信息结构。
- 右侧详情项允许展示一行次级信息，但不再形成“树里塞树”的拥挤感。
- 抽屉内不再使用“步骤 1 / 步骤 2 / 当前页面按钮”的向导语法。
- 按钮权限保留，但只在“当前页面”上下文中平铺展示。

## 前端实现边界

重点实现集中在前端：

- `spring-boot-iot-ui/src/views/RoleView.vue`
- `spring-boot-iot-ui/src/utils/menuAuth.ts`
- `spring-boot-iot-ui/src/components/role/*`
- 对应 Vitest 测试

需要补充的核心能力：

- 基于完整树计算节点祖先/后代关系
- 计算自定义半选态
- 过滤显示树但不破坏授权状态
- 从当前节点派生直属子级详情列表

- 已有角色 `menuIds` 能正确回显到新授权树
- 勾选父节点时，后代默认一并授权
- 取消部分子节点后，父节点显示半选态
- 未先勾选父节点时，仍可从右侧对子节点单独授权
- 保存请求继续只提交 `menuIds`
- 搜索过滤前后，既有勾选状态不丢失
