# 产品经营工作台紧凑总览改造设计

**Date:** 2026-04-05  
**Status:** Approved in-session for spec drafting  
**Audience:** `/products` 产品经营工作台前端实现 / 文档维护  
**Scope:** 在不改路由、不改后端接口的前提下，继续收紧产品经营工作台头部、总览页和契约字段页的视觉结构，使其更紧凑、更协调。

## 1. 背景

当前产品经营工作台虽然已经从早期重型版本收口，但仍存在 3 类问题：

1. 页头左侧内容与右侧指标区留白过多，不协调。
2. `契约状态 / 下一步建议` 占位过重，右侧 `关联设备` 卡的说明文字也过多。
3. `在线覆盖 / 30 日活跃 / 平均在线 / 档案摘要` 仍保留线性台账感，和用户期望的紧凑卡片总览不一致。

用户本轮明确要求：

1. 取消 `契约状态`、`下一步建议`，仅保留 `关联设备`。
2. `产品Key / 接入协议 / 节点类型 / 数据格式` 这一行字体与页头总描述保持同级。
3. 去掉不对称线条和橘色强调线。
4. `在线覆盖 / 30 日活跃 / 平均在线 / 档案摘要` 改为卡片表达。
5. `档案摘要` 中 `厂商 / 最近更新 / 补充说明` 要更自适应；其中 `补充说明` 超出时单行省略。
6. `关联设备` 页中的 `设备清册` 改名为 `设备清单`。
7. `契约字段` 页中的多余线条也一起取消。
8. 最终又补充确认：页头左侧内容要与右侧 `关联设备` 卡等高，右侧卡只保留 `关联设备 + 数量`，不再保留说明短句。

## 2. 目标

本轮目标：

1. 把页头收口为“左侧产品信息卡 + 右侧关联设备单卡”的双卡结构。
2. 让双卡等高，并显著减少头部留白。
3. 把产品总览从“横向线性台账”改为“卡片式经营总览”。
4. 保持档案信息可读，但避免长说明撑高整页。
5. 把 `关联设备 / 契约字段` 两个工作区中的旧线条语法一起清理，统一成更轻的卡片式工作台。

## 3. 非目标

本轮不做：

1. 不改 `/products` 路由结构。
2. 不改产品查询、详情、物模型、关联设备的后端接口。
3. 不改产品经营工作台的业务页签结构，仍保留 `产品总览 / 关联设备 / 契约字段`。
4. 不改契约字段抽屉的 compare/apply 行为，只处理视觉与文案层级。

## 4. 方案对比

### 方案 A：双卡紧凑头部，推荐

做法：

1. 左侧改成产品信息卡，包含 `产品经营页 / 产品名 / 一句短说明`。
2. 右侧只保留 `关联设备 + 数量` 单卡，不保留辅助说明。
3. 两张卡强制等高。
4. 下方总览改成 `3` 张经营指标卡 + `1` 张档案摘要卡。

优点：

1. 最符合用户“保留右侧关联设备，但去掉空白和冗余说明”的目标。
2. 与当前工作台结构兼容，改造集中在前端布局和样式。
3. 容易与下方总览卡形成统一语法。

缺点：

1. 需要同时调整头部、总览区和部分测试。

### 方案 B：完全扁平化头部

做法：

1. 左侧保留裸标题和说明。
2. 右侧只保留一个轻量数字块，不做卡。
3. 其余内容全部下沉到总览区。

不采用原因：

1. `关联设备` 会显得过轻，无法承担用户明确保留的右侧视觉锚点。
2. 更容易让下方总览区反而显得过重。

本轮采用 `方案 A`。

## 5. 最终设计

### 5.1 页头结构

页头改为双卡：

1. 左侧 `产品信息卡`
2. 右侧 `关联设备卡`

其中：

1. 左侧卡保留：
   - `产品经营页`
   - 产品名称
   - 1 句短说明
2. 右侧卡只保留：
   - `关联设备`
   - `X 台`
3. 删除：
   - `契约状态`
   - `下一步建议`
   - `关联设备` 卡中的描述短句
4. 左右两张卡高度对齐，避免左大面积留白、右卡过轻的问题。

### 5.2 页头文案

页头左侧短说明收口为 1 句短话，例如：

1. `当前已有运行设备，可继续核对字段。`

要求：

1. 不再使用长说明或判断墙。
2. 文案长度明显短于当前版本。

### 5.3 产品身份信息行

`产品Key / 接入协议 / 节点类型 / 数据格式` 这一行：

1. 保留在头部与页签之间。
2. 字号提升到与页头总描述同级，不再偏小。
3. 删除上下不对称分隔线。
4. 删除橘色强调线。
5. 改为更轻的正文信息行语法。

### 5.4 页签区

页签保留：

1. `产品总览`
2. `关联设备`
3. `契约字段`

调整：

1. 取消橘色激活下划线。
2. 改用更轻的文字高亮或中性色底部细线。
3. 整体与头部、总览卡的轻量风格保持一致。

### 5.5 产品总览

总览区改为两层：

1. 第一层：`在线覆盖 / 30 日活跃 / 平均在线` 三张等高卡
2. 第二层：`档案摘要` 单独一张卡

要求：

1. 不再使用横向线性分隔式台账。
2. 不再保留橘色线条。
3. 卡片间距收紧，避免总览区看起来过散。

### 5.6 档案摘要卡

档案摘要卡内固定 3 个字段：

1. `厂商`
2. `最近更新`
3. `补充说明`

布局规则：

1. `厂商` 和 `最近更新` 使用自适应宽度，按内容自然收口。
2. `补充说明` 占据剩余宽度。
3. `补充说明` 默认只显示 1 行。
4. 当内容超出时，使用省略号隐藏超出部分，不换行。

### 5.7 关联设备页

仅改表达：

1. 把 `设备清册` 改为 `设备清单`

本轮不改设备列表行为。

### 5.8 契约字段页

本轮只做轻量视觉清理：

1. 删除多余线条
2. 删除橘色强调线
3. 保持与产品总览一致的卡片化、轻描边语法

不改：

1. `待核对字段 / 正式字段` 结构
2. compare/apply 交互逻辑
3. 关系归一、自动提炼等业务流程

## 7. 验收点

1. 页头仅保留右侧 `关联设备` 单卡，且无说明短句。
2. 页头左侧内容与右侧卡等高。
3. `产品Key / 接入协议 / 节点类型 / 数据格式` 的字号与总描述一致。
4. 橘色线条和不对称分隔线全部移除。
5. `在线覆盖 / 30 日活跃 / 平均在线` 以卡片形式展示。
6. `档案摘要` 改为卡片，`厂商 / 最近更新` 自适应，`补充说明` 单行省略。
7. `关联设备` 页出现 `设备清单` 而不是 `设备清册`。
8. `契约字段` 页不再出现多余线条回流。
