# 设备资产中心详情抽屉扁平化改造设计

## 背景

`/devices` 当前详情抽屉仍是“重摘要卡 + 多段信息卡纵向堆叠”结构。字段虽然完整，但存在三个问题：

1. 首屏摘要权重过高，压住后续信息阅读。
2. 资产档案、拓扑、运维、认证、元数据拆得过碎，导致上下文来回跳转。
3. 已登记与未登记设备的详情语法差异较大，不利于资产中心统一维护。

本轮默认采用脑暴阶段推荐的 `A` 方案，即“顶部轻摘要带 + 中段双轴台账 + 底部补充区”。

## 目标

- 把设备详情抽屉调整为更扁平的控制台式工作台。
- 保留层次感，但去掉重复强调和重卡片堆叠。
- 让上下文按“身份/部署”与“运行/认证”形成对称阅读节奏。
- 未登记设备继续复用同一抽屉语法，只在信息分组上体现差异。

## 信息架构

### 已登记设备

1. `资产状态与接入概况`
   - 用 4 个轻摘要单元承接产品归属、在线状态、激活状态、设备状态。
   - 只表达当前判断和最近关键时间，不再承担完整档案职责。
2. `身份与部署台账`
   - 承接设备 ID、设备编码、设备名称、产品归属、节点类型、协议、固件、IP、部署位置。
3. `运行与认证台账`
   - 承接最近在线/离线/上报、创建/更新时间，以及 `Client ID / 用户名 / 密码 / 设备密钥`。
4. `关系与建档补充`
   - 承接父设备、网关设备及主键关系。
5. `扩展元数据快照`
   - 保留 `metadataJson` 的结构化预览，但下沉为补充层。

### 未登记设备

1. `资产状态与接入概况`
   - 用 4 个轻摘要单元表达登记状态、最近上报、失败阶段、失败摘要。
2. `来源档案与失败摘要`
   - 承接设备编码、产品标识、协议编码、来源记录、Topic、失败摘要。
3. `最近载荷`
   - 保留原始 payload 预览，不新增第二套失败详情壳。

## 组件与边界

- 新增独立组件：`spring-boot-iot-ui/src/components/device/DeviceDetailWorkbench.vue`
- `DeviceWorkbenchView.vue` 只负责：
  - `StandardDetailDrawer` 容器
  - 补数状态提示
  - 标题、副标题、标签与底部动作
- 设备详情字段排版、分组和样式全部下沉到 `DeviceDetailWorkbench`

这样可以与现有 `ProductDetailWorkbench` 路径对齐，避免设备页继续在超长视图文件里累积详情布局逻辑。

## 视觉语法

- 顶部摘要区改为浅色轻面板，不再用多张重卡片做英雄区。
- 中段使用两个等权账本面板，形成左右对称的主阅读区。
- 底部补充区压低视觉权重，只保留轻分隔和安静背景。
- 颜色继续复用品牌橙、蓝灰正文和共享描边 token，不引入私有主题色。
- 字段值优先使用平直台账节奏，不在详情页内部继续叠加说明墙和重复判断句。

## 测试与验证

- 新增 `DeviceDetailWorkbench` 组件测试，覆盖：
  - 已登记设备的双轴账本结构
  - 未登记设备的来源/载荷结构
  - 旧标题语义不回流
- 保持 `DeviceWorkbenchView` 现有页面级测试和列表回归测试可通过。
- 完成后执行：
  - `npm run test -- DeviceDetailWorkbench`
  - `npm run test -- DeviceWorkbenchView`
  - `npm run build`
  - `npm run component:guard`
  - `npm run list:guard`
