# 对象洞察重要指标配置设计

## 目标

让产品定义中心、产品经营工作台和契约字段页都能明确设置“哪些正式字段必须显示在对象洞察台的属性趋势预览中”，并确保最终只有一份正式真相位。

## 单一真相位

- 正式配置继续存放在 `iot_product.metadata_json.objectInsight.customMetrics[]`。
- 不新增平行表，也不在 `/insight` 设备页本地维护第二套状态。
- 契约字段页和产品编辑页都回写同一份产品级 `metadataJson`。

## 三个入口的职责

### 1. 产品定义中心

- 继续保留 `对象洞察配置` 作为正式编辑入口。
- 新增“从当前正式字段快速加入”能力，避免只能手工输入 identifier。
- 允许把正式字段快速设为 `监测数据` 或 `状态数据` 趋势重点指标。

### 2. 产品经营工作台

- 编辑治理页复用同一套对象洞察配置编辑器和同一份正式字段候选。
- 保存时仍由现有 `updateProduct` 写回产品 `metadataJson`。

### 3. 契约字段

- 在 `当前已生效字段` 的正式字段卡片中新增快捷动作：
  - `设为监测趋势`
  - `设为状态趋势`
  - `取消趋势展示`
- 点击后立即回写产品 `metadataJson`，不要求用户再切回编辑页额外保存。

## 运行时消费规则

- `/insight` 继续按“设备级覆盖 > 产品级正式配置 > 内置注册表 > 自动识别”读取配置。
- 这次新增一条更明确的趋势排序规则：
  - `includeInTrend=true && includeInExtension=false` 的正式重点指标优先排在趋势分组前面。
  - `enabled=false` 的正式配置要能从趋势分组里排除，即使它能被运行时自动识别，也不能继续展示。
- `sortNo` 作为正式重点指标的排序依据。

## 不做的事

- 不新增对象洞察专用后端表。
- 不新增对象洞察独立菜单或独立配置页。
- 不把事件/服务字段纳入趋势图，只处理正式属性字段。
