# 产品契约字段工作区页面收口设计

## 背景

`/products -> 契约字段` 当前同时保留了：

- 阶段总览轨道：`待输入样本 / 待确认识别结果 / 待提交审批 / 审批中 / 已发布 / 可回滚`
- 主流程占位标题：`样本输入 / 识别结果`
- 历史辅助占位标题：`发布批次与风险联动 / 当前已生效字段`
- 各区块内部的二次说明文案

在共享环境里，这些提示多数不会形成持续变化的状态反馈，反而会把真实可操作内容淹没，导致页面显得拥挤、重复、判断性文案过多。

## 目标

把页面收口为“只保留有动作或有结果的内容区块”，让用户打开后优先看到可执行主流程，而不是一整层不会提供额外价值的说明。

## 设计结论

### 方案选择

采用“保留主内容、删除重复导语”的最小收口方案：

1. 删除整排阶段轨道和“主流程区 / 历史辅助区”占位标题。
2. 保留真实有内容的区块：
   - 样本输入
   - 识别结果
   - 审批提交回执 / 正式发布结果
   - 当前已生效字段
3. 对仍需保留的标题说明做短句化，避免再出现“解释解释”的二层文案。
4. 保留版本台账、回滚试算等确有业务价值的结果区，但不再用额外的“发布批次与风险联动”壳层包一层。

### 页面结构调整

页面调整后保留以下阅读顺序：

1. 顶部摘要区：保留标题、主按钮、关键数字和回滚试算/版本台账等结果信息。
2. 主流程区：
   - 样本输入
   - 识别结果
   - 本次生效 / 审批回执
3. 正式结果区：
   - 当前已生效字段

### 文案收口规则

- 删除阶段式提示语和流程编号感文案。
- 删除“只支持……”“当前只展示……”这类默认重复提示；仅在存在边界约束或空态时保留必要说明。
- 审批与回执区域继续保留“审批状态 / 审批单 / 执行时间”等结果型信息，不保留额外解释层。

## 非目标

- 本轮不改变 compare/apply/审批/回滚 的业务流程。
- 不新增新卡片、新配色、新一级分区。
- 不重做版本台账、回滚试算、正式字段编辑交互。

## 影响范围

- 前端主文件：`spring-boot-iot-ui/src/components/product/ProductModelDesignerWorkspace.vue`
- 前端测试：`spring-boot-iot-ui/src/__tests__/components/product/ProductModelDesignerWorkspace.test.ts`
- 文档：`docs/02-业务功能与流程说明.md`、`docs/15-前端优化与治理计划.md`

## 验收口径

- 页面不再出现阶段轨道：`待输入样本 / 待确认识别结果 / 待提交审批 / 审批中 / 已发布 / 可回滚`
- 页面不再出现占位壳层：`发布批次与风险联动`
- 页面仍保留：`样本输入 / 识别结果 / 当前已生效字段`
- 审批、版本台账、回滚试算等真实结果区块仍可正常展示
- 页面首屏明显更紧凑，不再由提示语主导视觉层级
