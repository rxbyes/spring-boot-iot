# 产品编辑页简化设计

**Goal:** 将产品定义中心的编辑页收敛为和“新增产品”同级的基础表单结构，只保留必要的产品属性编辑能力，并继续保留“对象洞察配置”。

**Architecture:** 编辑页继续复用现有产品表单数据模型与提交链路，不新增路由，不改接口，不引入独立编辑工作台。页面结构对齐新增产品表单，按 `基础档案 -> 接入基线 -> 补充说明 -> 对象洞察配置` 组织，去掉编辑页当前的治理语境、说明墙和额外编辑头部。

**Tech Stack:** Vue 3, TypeScript, Element Plus, existing `StandardFormDrawer` / product form components.

---

## 背景

当前产品定义中心存在两套编辑体验：

- 新增产品：标准表单抽屉，结构直接。
- 编辑产品：`ProductEditWorkspace` + 经营工作台语境 + 额外说明文案 + 编辑头部。

两者能力接近，但视觉和信息架构不一致，导致编辑页显得更复杂。

## 设计目标

1. 编辑页视觉和结构对齐新增产品页。
2. 保留 `对象洞察配置`，不裁掉该能力。
3. 仅保留编辑必需差异，例如 `产品 Key` 禁用、按钮文案随状态变化。
4. 不改产品详情子路由、不改数据提交协议、不改物模型治理页。

## 设计结论

### 1. 页面结构

编辑页只保留四个区块：

- `基础档案`
- `接入基线`
- `补充说明`
- `对象洞察配置`

删除以下内容：

- 编辑治理标题
- 编辑说明墙
- 额外 inline 提示语
- 经营工作台式的编辑语境

### 2. 交互差异

编辑态与新增态共享同一套表单骨架，仅保留以下差异：

- `产品 Key` 在编辑态保持禁用
- 提交按钮文案为 `保存`
- 取消按钮文案为 `取消编辑`
- 仍复用现有更新接口与校验逻辑

### 3. 组件边界

推荐将新增与编辑共用的表单内容抽成一个共享表单片段，避免两份模板继续漂移。

候选边界：

- 共享基础档案表单
- 共享接入基线表单
- 共享补充说明表单
- 共享对象洞察配置区

`ProductEditWorkspace` 保留为编辑入口承载组件，但其内部内容应与新增表单同构，而不是继续维护独立的编辑工作台样式。

## 非目标

- 不新增编辑专用路由
- 不调整详情页子路由结构
- 不移除对象洞察配置
- 不修改后端产品新增/更新接口

## 影响范围

- `spring-boot-iot-ui/src/components/product/ProductEditWorkspace.vue`
- `spring-boot-iot-ui/src/views/ProductWorkbenchView.vue`
- `spring-boot-iot-ui/src/__tests__/components/product/ProductEditWorkspace.test.ts`
- 相关产品工作台视图测试

## 验证标准

- 编辑页不再出现旧的编辑治理头部和说明文案
- 编辑页与新增产品页的字段顺序和分区一致
- `对象洞察配置` 仍可见、可编辑、可提交
- 编辑提交行为与现有更新链路一致
- 现有新增产品流程不受影响
