# Product Workbench Director Journal A Design

**Date:** 2026-04-01

**Scope:** `spring-boot-iot-ui` `/products` 统一 `产品经营工作台`

## Goal

把当前产品经营工作台从“展陈型高级扁平”重构为用户已确认的 `A. 年报中枢式`。新稿必须保留统一抽屉与四视图信息架构，但视觉表达要像高端业务年报内页，而不是后台卡片墙、对称简报页或展签陈列页。

## Confirmed Direction

- 主风格：指挥舱刊物式
- 配色：继续沿用系统现有品牌色与中性色令牌，不新增私有色盘
- 首屏重心：标题与核心规模双核并重
- 骨架：上刊头、下矩阵
- 正文承载：细线刊栏，不走厚卡片
- 字体气质：标题带宋感，正文保持现有系统黑体节奏
- 推荐稿：`A. 年报中枢式`

## Visual Grammar

### 1. Unified Masthead

统一头部改为“刊头”而不是“展签”或“简报条”：

- 左侧：产品标题、短说明句
- 右侧：单个核心规模数字，不堆叠多张指标卡
- 下方：一行去重元信息
- 最下：细线页签

头部不再使用：

- `产品展陈` 一类展签语言
- 居中的政企简报式中轴标题
- 多张并列核心指标卡
- 重复展示产品名称 / productKey / 协议 / 节点类型 / 数据格式

### 2. Overview As Report Spread

`经营总览` 改为“年报展开页”：

- 第一层：主规模刊栏，左主值右辅值
- 第二层：契约基线线表
- 第三层：档案摘要附注

`在线覆盖 / 30 日活跃 / 平均在线` 继续留在核心规模区，但表达要像刊栏注记，不再像色块指标卡。

### 3. Governance Tabs As Journal Sections

- `物模型治理`：刊头 + 摘要条 + 治理清单 / 正式模型正文
- `关联设备`：台账标尺 + 设备清册
- `编辑治理`：修订页眉 + 修订条 + 表单正文

三个页签都必须收掉解释型副文，优先呈现标题、条目、事实字段和值。

## Color and Material Rules

只允许使用现有系统设计令牌：

- `var(--brand)`
- `var(--brand-light)`
- `var(--panel-border)`
- `var(--panel-border-strong)`
- `var(--text-heading)`
- `var(--text-secondary)`
- `var(--text-caption)`

允许继续使用：

- 极浅的品牌色混合底
- 极轻的放射或线性背景
- 细线边界

不允许继续回流：

- 大块私有米色 / 金棕色渐变
- 与系统品牌无关的“年报专用”新配色
- 多个同权重厚底块叠加

## Content Compression Rules

- 删除判断性文案和说明墙
- 删除重复字段展示
- 删除装饰性但无信息增量的副标题
- 同一层只保留一个主信息焦点

## Test Impact

现有产品工作台 5 个组件测试需要从“展陈型”合同切到“年报中枢式”合同，重点验证：

- 新刊头结构
- 新总览刊栏结构
- 新治理摘要条与治理清单
- 新设备清册标尺结构
- 新修订页眉与修订条

## Non-Goals

- 不改 `/products` 外层列表信息架构
- 不新增新接口
- 不改统一工作台四视图路由与状态保活策略
- 不引入私有设计系统或第三方字体依赖
