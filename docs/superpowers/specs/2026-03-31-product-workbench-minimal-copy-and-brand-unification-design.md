# 产品经营工作台极简文案与主色统一设计

> 日期：2026-03-31
> 范围：`spring-boot-iot-ui`
> 主题：压缩产品详情页解释文案，并把关联设备页的指标区统一回主色体系
> 状态：设计已确认，待实现

## 1. 背景

在“首长简报式”升级完成后，用户继续指出两个验收细节：

- `经营总览` 详情页里仍有较多解释型文案，页面读起来偏重
- `关联设备` 页指标带用了带明显区分的多色配色，和主色体系不统一

当前问题不在功能，而在收敛度：

- 详情页需要进一步压缩成“标题 + 必要结论”
- 关联设备页需要从“分彩卡片”回到“统一主色的经营台账”

## 2. 目标

- 把详情页压缩到极简政企风格，只保留有必要的判断、字段和值
- 去掉重复、副解释、说明墙式文案
- 把关联设备指标横带统一回主色体系
- 继续保留表格里的在线/离线、激活状态语义标签，不影响操作识别

## 3. 非目标

- 不改路由，不改接口，不改交互流程
- 不改变详情页和关联设备页的模块边界
- 不动物模型治理、编辑治理页的结构
- 不移除表格里的语义状态 tag

## 4. 已确认方向

用户已确认：

- 详情页文案压缩程度：`极简`
- 关联设备页配色方案：`全主色`

落地解释固定为：

- 详情页每个板块尽量只保留 `标题 + 1 条必要结论`
- 关联设备页指标区全部回到主色系浅底和主色边框
- 语义色只留在表格行内状态标签中

## 5. 详情页设计

### 5.1 删除项

从 `ProductDetailWorkbench.vue` 中删除或压缩以下说明型内容：

- 章节标题下面的长句副说明，尽量缩为一句或删除
- `趋势摘要 / 接入提示` 下的第二层副文
- `接入契约摘要 / 治理摘要` 这类额外解释卡
- `当前建议` 下的长段落说明

### 5.2 保留项

以下内容继续保留：

- `经营主判断`
- 主指标与 3 个辅助指标
- 趋势摘要和接入提示的结论句
- 接入契约字段和值
- 产品档案字段和值
- 当前建议
- 维护规则、变更前确认清单

### 5.3 目标形态

详情页最终应更接近：

- 一级：主舞台
- 二级：必要结论
- 三级：字段和值

而不是：

- 标题
- 解释
- 解释的解释
- 摘要卡
- 摘要卡说明

## 6. 关联设备页设计

### 6.1 配色调整

指标横带的 4 个卡块统一调整为：

- 主色浅底
- 主色轻边框
- 中性文字层级

不再使用：

- `success`
- `danger`
- `accent`

作为整块卡片的底色/边框来源。

### 6.2 保留项

表格中的以下局部状态继续保留语义色：

- 在线 / 离线
- 已激活 / 未激活

因为这属于行内状态识别，而不是页面主视觉。
