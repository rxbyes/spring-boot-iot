# 产品经营工作台去重收敛设计

> 日期：2026-04-04
> 范围：`spring-boot-iot-ui`
> 主题：继续收敛 `/products` 产品经营工作台中的重复规模、契约、档案与治理说明
> 状态：设计已确认，待实现

## 1. 背景

产品经营工作台已经完成“经营总览 / 物模型治理 / 关联设备 / 编辑治理”四个视图的统一抽屉承载，但当前仍存在几个影响验收观感的问题：

- 同一组规模、契约、档案信息在抽屉头部、经营总览和子工作区被重复表达
- 运行期判断语句偏长，且与页面主目标“看规模、看契约、看档案”重复
- `关联设备` 与 `编辑治理` 仍保留了重复统计条，稀释真正的主工作区内容
- `物模型治理` 的标题、副标题与步骤说明存在同义反复

当前问题不在能力缺失，而在信息密度和职责边界没有完全收住。

## 2. 目标

- 删除抽屉头部与子视图中的重复表达，保留唯一主入口信息
- 让契约信息在头部一次说清，不再在总览左下角再拆一遍
- 让 `在线覆盖 / 30 日活跃 / 平均在线` 和 `档案摘要` 都回到单行台账节奏
- 收短 `物模型治理` 的说明文案，只保留必要治理语义
- 移除 `关联设备` 和 `编辑治理` 中重复统计，突出真正操作区

## 3. 非目标

- 不改路由，不新增页面，不改接口，不改数据来源
- 不改变产品经营抽屉的四视图结构
- 不重做正式模型治理流程，不引入新的治理模式
- 不调整列表页 `/products` 的主表结构与行内动作

## 4. 已确认方向

用户已确认采用“最小收敛”方案：

- 只删除重复文案、重复统计和重复契约拆项
- 不做大重排，不引入新的页面层级
- 以“言简意赅”为唯一文案尺度

## 5. 设计方案

### 5.1 抽屉头部

- 删除运行期判断文案中 `已进入运行期，当前继续围绕规模、契约与档案校准。` 一类长句
- 头部继续保留右上角 `核心规模`，作为整个工作台唯一的总量主指标
- 头部契约线改为单句直出：
  - `产品key：nf-collect-rtu-v1｜接入协议：mqtt-json｜节点类型：直连设备｜数据格式：JSON`
- 当字段缺失时仍使用 `--`，但保持同一句式，不回退成多点分散显示

### 5.2 经营总览

- 删除与头部重复的 `核心规模 / 关联设备总量`
- 删除左下角“契约基线”中拆开的重复契约项，避免和头部再次重复
- `在线覆盖 / 30 日活跃 / 平均在线` 保持同一行横排
- `档案摘要` 改成单行横排，保留：
  - 厂商
  - 最近更新
  - 补充说明

### 5.3 物模型治理

- 继续保留当前 `手动提炼 / 正式模型` 双模式
- 收短标题、副标题和步骤说明，避免“治理”“契约”“证据”在一屏内反复重复
- 保留真正必要的信息：
  - 当前入口
  - 治理语义
  - 正式模型数量
  - 进入治理的主按钮

### 5.4 关联设备

- 删除顶部统计尺：
  - 设备总数
  - 在线设备
  - 离线设备
  - 在线比例
- 保留 `设备清册` 作为主阅读面，避免再次复述已在总览和头部表达过的规模信息

### 5.5 编辑治理

- 删除顶部 `产品 Key / 协议编码 / 节点与状态` 统计尺
- 保留必要说明和表单本体
- 维持 `基础档案 / 接入基线 / 补充说明` 三段编辑结构不变

## 6. 测试与文档

- 先写组件测试，锁定以下收敛结果：
  - 抽屉头部使用完整契约句式
  - 总览不再重复显示契约拆项和核心规模
  - 关联设备页无重复统计条
  - 编辑治理页无重复统计条
  - 物模型治理文案明显收短
