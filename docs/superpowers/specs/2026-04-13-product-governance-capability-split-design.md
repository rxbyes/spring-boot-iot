# 产品治理能力分流设计

## 背景

当前 `/products` 与 `契约字段` 工作区把“是否存在可入目录字段”直接等同于“后续风险治理是否适用”。这会把采集型、预警型、视频型产品错误收口成“目录发布、风险点绑定与策略覆盖暂不适用”，从而遮蔽其真实支持的“设备级风险点绑定”能力。

产品现状已明确分为四类：

- `MONITORING` 对应监测型，具备测点治理能力
- `COLLECTING` 对应采集型，不进入目录/阈值治理，但支持设备级风险点绑定
- `WARNING` 对应预警型，不进入目录/阈值治理，但支持设备级风险点绑定
- `VIDEO` 对应视频型，不进入目录/阈值治理，但支持设备级风险点绑定
- `UNKNOWN` 表示能力待确认，不能再被写成“暂不适用”

## 目标

1. 让产品工作台按“产品能力”而不是 `publishableContractPropertyCount` 决定后续治理链路。
2. 为产品补齐正式能力真相源，避免所有分流都靠关键词猜测。
3. 保持监测型现有链路不退化，同时让非监测型产品显式暴露“去风险点绑定”入口。

## 真相源

产品能力真相源统一落在 `iot_product.metadata_json`：

```json
{
  "governance": {
    "productCapabilityType": "COLLECTING"
  }
}
```

解析优先级：

1. `metadataJson.governance.productCapabilityType`
2. 兼容现有稳定字段（若后端未来显式回传产品能力，可直接接入同一解析器）
3. 产品名称关键词兜底

`UNKNOWN` 的业务语义固定为“能力待确认”。

## 页面行为

### `/products` 页面 notices

- `MONITORING`
  - 保留现有主链路：`合同发布 -> 风险指标目录 -> 风险点绑定 -> 阈值策略`
  - 继续展示待发布目录、待绑定风险点、待补阈值策略等治理待办
- `COLLECTING / WARNING / VIDEO`
  - 保留合同治理语义
  - 不再生成目录发布、待绑定风险点数量、待补阈值策略三类待办
  - 额外展示能力说明：当前产品不进入风险指标目录与阈值策略治理，但支持设备级风险点绑定
  - 提供 `去风险点绑定` 入口，直接跳到 `/risk-point`
- `UNKNOWN`
  - 不显示“暂不适用”
  - 展示“产品能力待确认”
  - 提供 `去完善产品能力` 入口，回到产品经营工作台的 `编辑档案`

### `/products -> 契约字段`

- `MONITORING`
  - 继续展示四步治理卡：`合同发布 / 风险指标目录 / 风险点绑定 / 阈值策略`
- `COLLECTING / WARNING / VIDEO`
  - 改为能力感知卡片：
    - `合同发布`
    - `版本台账`
    - `设备级风险点绑定`
  - 额外说明：目录发布与阈值策略仅监测型适用
- `UNKNOWN`
  - 改为：
    - `合同发布`
    - `版本台账`
    - `能力待确认`
  - 额外说明：请先完善产品能力，再决定是否进入风险目录或设备级绑定治理

## 编辑档案

`编辑档案` 增加 `产品能力` 选择项，候选值固定为：

- `监测型`
- `采集型`
- `预警型`
- `视频型`
- `待确认`

保存时写回 `metadataJson.governance.productCapabilityType`，并与既有 `objectInsight` 共存。

## 风险点绑定兼容

`riskPointDeviceBindingCapability` 同步补齐 `COLLECTING`，并将其视为“不支持测点级绑定、只支持设备级绑定”，保证后续风险绑定工作台与产品能力口径一致。

## 测试范围

1. 产品能力解析工具：显式元数据、关键词兜底、未知能力
2. `/products`：
   - 监测型继续展示全链路待办
   - 采集型不再显示目录/绑定数量/策略待办，但显示设备级绑定说明
   - 未知能力显示“能力待确认”与完善入口
3. `契约字段`：
   - 采集型显示 `设备级风险点绑定`
   - 不再显示“风险点绑定暂不适用 / 阈值策略暂不适用”
4. `编辑档案`：
   - 保存时把产品能力正确写入 `metadataJson.governance`
5. 风险绑定能力工具：
   - `COLLECTING` 走设备级绑定语义
