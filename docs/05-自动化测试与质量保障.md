# 05 自动化测试与质量保障

更新时间：2026-03-20

## 1. 测试策略总览

当前项目采用“单元测试 + 协议回归 + 真实环境验收脚本”的组合策略。

- 单元测试：覆盖核心服务逻辑、协议解析、安全校验、审计与通知。
- 协议回归：重点覆盖 MQTT Topic 解析、加密解密、签名防重放、二进制帧解析。
- 真实环境验收：以 `application-dev.yml` 所指向环境为基线，验证端到端业务可用性。

## 3. 命令与执行方式

### 3.1 Maven 测试

```bash
mvn -s .mvn/settings.xml test
```

### 3.2 构建验证

```bash
mvn -s .mvn/settings.xml clean package -DskipTests
```

### 3.3 真实环境验收（推荐）

- 后端：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-backend-acceptance.ps1
```

- 前端：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/start-frontend-acceptance.ps1
```

- 验收场景基准：`docs/test-scenarios.md`。

## 4. 已知问题与环境差异

1. `DeviceMessageServiceImplTest` 在部分 JDK 17 环境可能因 Mockito inline agent 自附加失败而报错。
2. `scripts/start-backend-acceptance.ps1`、`scripts/start-frontend-acceptance.ps1`、`scripts/run-business-function-smoke.ps1`、`scripts/run-business-function-browser.ps1` 已改为根据脚本位置自动推导仓库根目录，不再依赖固定 Windows 盘符路径。
3. `【待确认】` 当前仓库未见统一 CI 配置（如 GitHub Actions/Jenkinsfile），自动化执行口径主要依赖本地或共享环境。

## 5. 质量门禁建议（当前执行口径）

每次需求合并前建议至少完成：

1. `mvn -s .mvn/settings.xml clean package -DskipTests`。
2. 受影响模块单元测试。
3. 关键链路手工验收（HTTP 上报、MQTT 上报、告警/事件闭环、权限登录）。
4. 文档同步更新（核心文档 `01`~`08` + README）。
