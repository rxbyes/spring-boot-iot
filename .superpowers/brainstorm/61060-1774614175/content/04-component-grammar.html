Hero 中枢头部

每页都用同一套舞台头部，但强调“本页判断”和“本页动作”，不再重复做导航首页。

2. 指标与提醒层

指标与提醒层

统一用“1 张主提醒卡 + 3 到 4 张指标卡”的节奏，避免每页都是平铺的小统计块。

3. 主工作区

辅战区: 时间线 / 原始响应 / 最近会话 / 诊断建议

主工作区

资产页以左侧主台账为主，诊断页以右侧结果舞台为主，但两者都仍在同一套工作区语法下。

4. 详情 / 抽屉语法

治理建议 / 联动动作

详情 / 抽屉语法

统一采用“摘要秒开 + 分区字段卡 + 建议动作”的抽屉体验，保持产品、设备、trace、异常详情都属于同一家族。

统一的部分

  • 品牌橙主动作、云蓝辅助态、蓝灰正文
  • Hero / 指标卡 / 主工作区 / 抽屉 这四层固定存在
  • 统一复用 StandardWorkbenchPanel、PanelCard、MetricCard、StandardDetailDrawer 等现有能力

变化的部分

  • 资产页更稳，列表密度更高
  • 诊断页更强调结果舞台和联动排查
  • 总览页更强调跨模块分发，不承接重明细
