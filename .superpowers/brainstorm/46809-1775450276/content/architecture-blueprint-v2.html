方案 B 修订版：双核心域总体蓝图

不是“物模型治理完成后顺手接风险”，而是同时建设两个核心域：语义契约治理域 与 风险运营闭环域。

核心域 A：语义契约治理域
规范语义层

行业/业务规范字段、命名规则、单位、监测类型、是否允许进风险闭环。

运行证据层

$dp 报文、解密结果、消息日志、latest、逻辑通道、父子设备关系、协议模板证据。

治理决策层

差异比对、命名推荐、冲突裁决、缺失识别、样板扩展、版本治理。

合同发布层

iot_product_model 只保存“已发布正式合同”，不再承载全部治理过程。

核心域 B：风险运营闭环域
风险对象层

风险点、对象档案、设备/测点正式绑定，只认正式规范 identifier。

策略决策层

阈值策略、判级策略、联动编排、预案匹配、策略版本和启停。

事件闭环层

告警记录、事件协同、工单流转、应急处置、处置审计。

运营呈现层

实时监测、GIS 态势图、对象洞察、告警运营、运营分析。

两大核心域的唯一耦合点

语义契约治理域输出： 正式规范字段合同（产品级）。

风险运营闭环域消费： 已发布的规范 identifier、可用于风险的字段标签、字段分组与单位。

明确不再直接消费： 厂商原始字段名、$dp 逻辑通道名、临时 compare 候选。

主链路视角

接入主链路： Topic / 解密 / 解码 / 关系归一 / 标准化证据沉淀

治理主链路： 规范库对齐 / 差异比对 / 人工裁决 / 合同发布

风险主链路： 风险点绑定 / 阈值策略 / 联动编排 / 应急预案 / 告警运营
