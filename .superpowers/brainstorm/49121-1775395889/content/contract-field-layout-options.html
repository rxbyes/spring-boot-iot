契约字段页收口方案

目标：取消二层抽屉，只保留手动样本提炼，契约字段在同一页完成输入、提炼、确认和查看正式字段。

A. 单页纵向工作台（推荐）
产品经营工作台 / 契约字段
顶部：产品摘要 + 已生效 / 本次识别 / 待生效
样本类型：业务数据 / 状态数据 / 单台设备 / 复合设备
JSON 输入区（支持粘贴后自动格式化）
识别结果列表
本次生效
当前已生效字段

单页纵向工作台

最接近“在契约字段同一页完成全部动作”。阅读顺序直白，删掉抽屉后最自然。

优点

  • 最容易理解
  • 移动端适配最稳
  • 最适合彻底删除自动提炼链路

代价

  • 页面会更长
B. 左输入右结果
产品经营工作台 / 契约字段
左侧
样本类型
JSON 输入区
右侧
识别结果
本次生效
底部：当前已生效字段

左右并排编辑台

桌面端效率高，输入和结果能同时看到，但移动端需要明显折叠。

优点

  • 桌面端紧凑
  • 输入与结果并读效率高

代价

  • 移动端更难收口
  • 和当前工作台单栏语法有点偏离
C. 单页步骤板
产品经营工作台 / 契约字段
步骤 1：粘贴上报 JSON
步骤 2：选择识别结果
步骤 3：确认生效
步骤 4：正式字段

单页步骤板

仍然是同一页，但强调流程感。适合培训式产品，不够像你想要的“精简直接”。

优点

  • 引导最强

代价

  • 仍然有流程墙感
  • 不够利落
